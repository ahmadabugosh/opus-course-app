# Using Opus Templates

## Why Use Templates?

Instead of building workflows from scratch, Opus provides a library of **ready-made templates** for common business processes. These pre-built workflows can save you dozens of hours per month and help you 10x your productivity right from day one.

## Explore the Template Library

**Start here:** Visit the [Opus Templates Library](https://app.opus.com/app/workflow/create/templates) to browse hundreds of production-ready workflows across different industries and use cases.

## What You'll Learn
- How to find templates for your specific needs
- How to review and customize template workflows
- How to fix action items before activation
- How to activate and test your workflow

## Key Concepts
- Template library and search
- Action Items section
- Workflow activation and testing
- Industry-specific templates

## Step-by-Step Guide

### 1. Browse the Template Library
Visit the [Templates page](https://app.opus.com/app/workflow/create/templates) and explore the available workflows. You can:
- Search by workflow name
- Filter by industry (Aerospace, Agriculture, Utilities, etc.)
- Filter by creator (Opus-verified templates)

### 2. Find a Relevant Template
Look for a template that matches:
- A process you currently run manually in your job
- A workflow you've been wanting to automate
- An industry-specific task relevant to your work

Examples: Content compliance validation, aircraft transition management, allergen testing reporting, emergency response plans, asset upgrade requests.

### 3. Review the Template Details
Click on a template to see:
- Full workflow description
- Industry classification
- Input and output requirements
- Tags and labels
- Owner information

### 4. Use the Template
Click **"Use workflow"** to add it to your organization. The workflow will be copied to your account in draft mode.

### 5. Check Action Items
Before you can activate the workflow, check the **"Action Items"** section (shown with a red warning indicator). This section lists any required configurations that need to be fixed, such as:
- Missing workflow inputs (variables that need to be set)
- Invalid field mappings
- Required integrations or credentials

Click **"Fix"** next to each action item to resolve it.

### 6. Fix All Pending Issues
Work through each action item:
- Set required input variables
- Configure any missing mappings
- Add credentials for integrations (if needed)

Once all action items are resolved, the workflow is ready to activate.

### 7. Activate Your Workflow
Switch the workflow status from **"Inactive"** to **"Active"** using the toggle in the top right corner.

### 8. Test Your Workflow
Run a test job with sample data to ensure the workflow executes correctly. Monitor the output and verify the results match your expectations.

## Challenge

Find a template workflow that's relevant to your work or interests, customize it by fixing all action items, activate it, and run a successful test. Submit either:
- Your activated workflow URL, or
- Screenshot showing the activated workflow with action items resolved

## Tips & Best Practices
- Start with Opus-verified templates (marked with the Opus badge)
- Read the full workflow description before using it
- Test with realistic sample data that matches your use case
- Keep workflows in draft mode until all action items are fixed
- Document any customizations you make for future reference

## Real-World Impact

By using templates instead of building from scratch, you can:
- **Save 10-20 hours** per workflow
- **Launch automation faster** (days instead of weeks)
- **Learn best practices** from production-proven workflows
- **Focus on customization** instead of foundational setup
